# Projeto-DA-2

This project aims to analyse the Travelling Salesperson Problem (TSP) and design heuristics to solve it, using several datasets from the context of ocean shipping and urban deliveries. 

---------------

## Note
Due to their large size, and potentially storage issues, the real-world graphs are not fully connected.
As such, and only whenever you need to determine the distance between two nodes whose edge is not
provided, you may use the geographic latitude and longitude distance to determine â€œon demandâ€ the distance between two nodes and assume they are therefore connected. You will get a tour that is not â€œrealâ€
in the sense that the edges it is based on do not necessarily exist, but at least you guarantee the 2-
approximation bound. In order to compute the distances between nodes, you need to use the Haversine
distance computation described in appendix A.

## Backtracking Algorithm - T2.1 (4 points)
- [x] Develop a backtracking algorithmic approach to the TSP for a graph starting and ending the node tour on node labelled with the zero-identifier label
- [x] You are expected to use the small graphs to validate this approach and to observe that for the larger graphs this approach is not really feasible.
- [x] To this extent, you are suggested to plot the execution time (or other performance metrics you find significant) to illustrate the feasibility or not of this approach for larger graphs.

## Triangular Approximation Heuristic - T2.2 (4 points)
- [x] Use the geographic node data, and implement the approximation algorithm for TSP that relies on the triangular inequality to ensure a 2-approximation algorithm for this problem again starting and ending the node tour on node with the zero-identifier label
- [x] Compare with the backtracking algorithm for the same small graphs.
- [x] Add the missing edges on the toy graphs by calculating their distance.

## Other Heuristics - T2.3 (6 points)
- [x] Develop your own â€œefficientâ€ heuristic for TSP that can leverage, or not, the geographic node data.  The emphasis in this approach should be on efficiency as this algorithm should be feasible even for the large graphs.
- [x] Make use of several algorithmic techniques:
- [x] Divide-and-conquer to split the graph into multiple geographic sections
- [x] Use of clustering
- [x] Or both, possibly even in combination with the 2-approximation algorithm.
- [x] Note though, that in this algorithm the tour has to rely exclusively on â€œrealâ€ or existing edges and cannot be â€œapproximatedâ€ as you possibly have done in the 2-approximation algorithm. Nevertheless, you may use the â€œnon-realâ€ approximation solution as a basis for your heuristic.
- [x] Compare the quality and run-time performance of your heuristic solution against the 2-approximation algorithm (if you do not rely on it).
- [x] Although possible, you are not expected to present multiple heuristics, but rather a heuristic that works well over the basic 2- approximation or one that offers comparable tour length results but with noticeably faster execution time (i.e., time complexity).
